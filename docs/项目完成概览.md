## 🚀 下一步启动说明

### 1. 安装依赖

```bash
# 前端
cd frontend
npm install

# 后端
cd backend
npm install
```

### 2. 配置环境变量

```bash
# 前端
cp frontend/.env.example frontend/.env

# 后端
cp backend/.env.example backend/.env
```

### 3. 设置数据库

```bash
# 创建 PostgreSQL 数据库
sudo -u postgres psql
CREATE DATABASE toolibox;
CREATE USER toolibox_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE toolibox TO toolibox_user;
\q

# 运行迁移
cd backend
npx prisma migrate dev
npx prisma generate
```

### 4. 启动服务

```bash
# 终端 1 - 后端
cd backend
npm run dev

# 终端 2 - 前端
cd frontend
npm run dev
```

### 5. 访问应用

- 前端: http://localhost:3000
- 后端 API: http://localhost:8000/api/health

## ⚠️ 注意事项

### 开发环境
1. **Node.js 版本**: 需要 Node.js 20+
2. **PostgreSQL**: 需要安装 PostgreSQL 14+
3. **Cloudflare R2**: 需要配置 R2 凭证才能使用备份功能

### 首次运行
1. 必须先配置 `.env` 文件
2. 必须先运行数据库迁移
3. 建议先启动后端再启动前端

### 生产部署
1. 需要配置域名和 SSL 证书
2. 使用 PM2 管理后端进程
3. 配置 Nginx 反向代理
4. 设置防火墙规则

## 📊 数据库 Schema

```sql
Table: feedback
├── id (SERIAL PRIMARY KEY)
├── name (VARCHAR(100))
├── email (VARCHAR(100) NOT NULL)
├── subject (VARCHAR(200))
├── message (TEXT NOT NULL)
├── tool_name (VARCHAR(100))
├── status (VARCHAR(50) DEFAULT 'pending')
├── user_agent (TEXT)
├── ip_address (VARCHAR(45))
├── created_at (TIMESTAMP DEFAULT NOW())
└── backed_up_to_r2 (BOOLEAN DEFAULT FALSE)

Indexes:
- idx_feedback_status
- idx_feedback_tool_name
- idx_feedback_created_at
```
